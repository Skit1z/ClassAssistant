# 上课摸鱼搭子开发指南

### v1.0.2 优化点
- **UI 界面瘦身**：缩小整体尺寸，调整按钮比例为 8:2，视觉更协调、低调。
- **设置面板优化**：采用响应式布局，解决保存按钮被遮挡问题，支持前端外观（主题、圆角、透明度）即时微调。
- **背景主题联动**：优化 Ocean/Forest 等主题色，各组件背景与按钮颜色实现全局同步。
- **交互逻辑改进**：增加暂停/继续功能，优化窗口展开/收起的动画与尺寸控制。
- **错误兼容处理**：修复部分 Windows 终端环境下启动时的资源加载报错。

## 项目概述

上课摸鱼搭子是一个 Windows 桌面悬浮窗应用，目标是把课堂监听、点名预警、AI 救场和课后总结串成一条完整链路。

当前版本已经把最近几轮迭代里最容易出问题的几部分补齐了：

- 流式 ASR 重复写盘治理
- 课堂长转录的滚动摘要压缩
- 开发态与打包态的后端残留进程清理
- 启动监控前的课程名与资料选择
- 前端内置 .env 设置编辑面板

## 当前监控链路

### 1. 开始监控

前端点击“开始摸鱼”后不再直接调用 start_monitor，而是先打开 StartMonitorPanel：

- course_name：课程名称，可为空
- cite_filename：从 data/cite 中选一份资料，可为空

后端 monitor_router 会在启动前调用 TranscriptService.activate_cite_file，把选中的 cite 文本复制为 data/current_class_material.txt。

### 2. 转录写盘策略

MonitorService 现在区分本地 ASR 与在线流式 ASR：

- Local ASR：保持按句回调，识别到一句就追加一行。
- Seed-ASR / 其他流式：只落盘最终稳定句，partial 文本仅保存在内存。

核心处理点：

- _is_meaningful_text：过滤短碎片和纯标点
- _is_near_duplicate_locked：过滤重复句和相似修正句
- _append_entry_locked：统一收口写入前的清洗逻辑

### 3. 滚动摘要

当 _summary_source_entries 累积到 50 条后：

1. 调用 LLMService.compress_monitoring_progress
2. 把旧内容压缩进“历史摘要”
3. 在 class_transcript.txt 中保留：

```text
=== 历史摘要 开始 ===
...
=== 历史摘要 结束 ===
```

TranscriptService.get_recent_transcript 会自动跳过这段摘要块，只返回近期真实课堂记录供救场与进度总结使用。

## ASR 说明

### Local ASR

LocalASR 已调整以下参数以提升按句返回速度：

- energy_threshold = 220
- pause_threshold = 0.45
- phrase_threshold = 0.2
- non_speaking_duration = 0.25
- phrase_time_limit = 10

### Seed-ASR

Seed 的 bigmodel_async 会持续回传修正中的文本，因此不能直接把 result.text 当成最终记录。

当前实现采用：

- show_utterances = true
- result_type = single
- end_window_size = 800
- force_to_speech_time = 1000

只处理 utterances 中 definite = true 的稳定分句，并通过 start_time、end_time、text 组合键去重。

另外，_parse_server_response 会忽略空 payload 和非 JSON payload，避免再出现 JSONDecodeError 把线程打崩。

## 资料上传与引用

上传接口仍然是 POST /api/upload_ppt，但行为已调整：

- 解析后的文本不再直接覆盖 current_class_material.txt
- 会写入 data/cite/<safe_name>_<timestamp>.txt
- 前端收到 cite_filename 后刷新可选资料列表

这使得一个项目目录可以同时保留多份课程资料，真正开始监听时再决定用哪一份做 LLM 辅助上下文。

## 调试建议

### 后端健康检查

- GET /api/health
- GET /api/check_mic
- GET /docs

### 常见排查顺序

1. 先确认是否用了 api-service/.venv。
2. 再确认 8765 端口没有被旧后端占用。
3. 如果是打包问题，查看 release/data/_startup.log。
4. 如果是 Seed-ASR 问题，优先看 definite utterance 是否正常产生。
5. 如果是救场或进度总结异常，检查 class_transcript.txt 与 current_class_material.txt 是否按预期更新。

## 维护建议

- data/cite 属于运行时数据，不应提交到仓库。
- docs/img 下的 GIF 属于产品演示资源，应随 README 一起维护。
- 修改监控链路时，优先保证 Local ASR 与 Seed-ASR 逻辑分离，不要把流式修正文处理强行套给本地按句模式。
- 修改 build.ps1 或 backend.spec 后，最好完整跑一次 ./build.ps1 验证健康检查步骤。
    │       ├── MockASR         (ASR_MODE=mock)
    │       ├── DashScopeASR    (ASR_MODE=dashscope)
    │       │     └── dashscope.Recognition + pyaudio 录音线程
    │       └── SeedASR         (ASR_MODE=seed-asr)
    │             └── websocket-client + 二进制协议 + 收发分离线程 + pyaudio 录音线程
    │
    ├── _on_local_asr_text(text, is_final)
    │     ├── 本地 ASR 每句直接追加到转录文件
    │     └── 仅检查最近两行关键词，避免历史重复触发
    │
    └── _on_asr_text(text, is_final)
          ├── 计算增量 delta（只对新文字做关键词检测）
          ├── _write_transcript()
          │     ├── 句中：原地覆盖活动行（seek + truncate）
          │     └── 句末：固定该行，开始新活动行
          ├── _check_last_lines_keywords()
          └── _broadcast_alert(alert) → WebSocket → 前端 AlertOverlay
```

#### 转录文件实时更新策略

- 在线 ASR（Seed-ASR / DashScope）会实时更新转录文件中的**活动行**（seek + truncate 原地覆盖）
- Local ASR 按句返回结果，每句直接追加一行，避免本地识别结果互相覆盖
- 遇到句末标点（`。？！；`）时固定活动行，开始新的活动行
- 关键词检测只检查**最近两行文本**，避免历史文本重复触发
- 停止监控时自动将未固定的活动行写入

#### 关键词配置

关键词从 `data/keywords.txt` 加载，每行一个，`#` 开头为注释：

```
# 监控关键词列表
点名
随机
抽查
回答
签到
```

- 启动监控时自动重载
- 运行中可调用 `POST /api/reload_keywords` 热重载
- 也可通过 `POST /api/update_keywords` 临时追加自定义关键词

## 打包发布

### 一键打包（推荐）

```powershell
.\build.ps1 v1.0.0
# 或
build.bat v1.0.0
```

脚本自动完成：
1. 同步版本号到 tauri.conf.json / package.json / Cargo.toml
2. PyInstaller 打包后端（自动清理 PATH 中的 tesseract 避免损坏 DLL）
3. Tauri 编译前端（`--no-bundle`，跳过 MSI/NSIS）
4. 组装 `release/` 目录 + 生成启动脚本
5. 启动打包后端做健康检查验证
6. 压缩为 `ClassAssistant-vX.X.X-win-x64.zip`

### 单独打包前端 (Tauri → .exe)

```bash
cd app-ui
npx tauri build --no-bundle
```

产物路径：`app-ui/src-tauri/target/release/app-ui.exe`

### 单独打包后端 (PyInstaller → .exe)

```bash
cd api-service
.venv\Scripts\pyinstaller.exe backend.spec --clean --noconfirm
```

产物路径：`api-service/dist/class-assistant-backend/`

> 注意：`backend.spec` 中 `console=False`，后端 exe 将以无窗口模式运行。如需调试打包版，临时改为 `console=True` 即可看到日志输出。

### 发布包结构

```
ClassAssistant-vX.X.X/
├── 启动.bat                         # 一键启动脚本（后端后台运行，无多余窗口）
├── 上课摸鱼搭子.exe                  # Tauri 前端桌面窗口 (约 8.5 MB)
├── backend/                         # 后端服务目录
│   ├── class-assistant-backend.exe  # FastAPI 后端 (console=False, 无窗口)
│   ├── .env.example                 # 环境变量模板
│   └── _internal/                   # PyInstaller 运行时依赖（勿删除）
└── data/                            # 运行时数据
    └── summaries/                   # 课后笔记输出目录
```

### 用户安装使用步骤

1. **解压** zip 到任意目录（路径不要包含中文和空格最佳）
2. **配置** 首次双击 `启动.bat`，会自动创建 `backend/.env` 并打开记事本
3. **填写** API 密钥（ASR + LLM），保存后重新运行 `启动.bat`
4. **使用** 屏幕角落出现悬浮窗 → 上传课件 → 开始摸鱼

> 若仅测试 UI，将 `.env` 中 `ASR_MODE` 设为 `mock` 即可，无需真实 API 密钥。

### 常见安装问题

| 问题 | 解决 |
|------|------|
| 后端异常退出 | 检查 `backend/.env`；或在 spec 中改 `console=True` 重新打包查看日志 |
| 误连到开发环境后端 | 启动发布包前关闭 `dev.bat`；当前启动脚本也会自动清理 8765 端口占用进程 |
| 发布包数据没写到解压目录 | 检查解压目录下 `data/_startup.log` 与 `/api/health` 的 `data_dir` 返回值 |
| Windows Defender 拦截 | PyInstaller/Tauri 打包产物会被误报，添加信任即可 |
| 无法录音 | Windows 设置 → 隐私 → 麦克风 → 允许桌面应用访问 |
| 前端打不开 | 确认后端已启动，访问 http://127.0.0.1:8765/docs 验证 |
| libfribidi-0.DLL 错误 | tesseract 的损坏 DLL 被加载；启动脚本已自动清理 PATH；或手动卸载/修复 tesseract |
